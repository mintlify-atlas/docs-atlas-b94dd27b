---
title: Plan Server API
description: API endpoints for the plan review server
---

The Plan Server provides endpoints for reviewing and approving/denying plans submitted by Claude Code. It handles plan retrieval, version history, annotations, integrations, and decision handling.

## Server Configuration

The plan server uses the following environment variables:

- `PLANNOTATOR_REMOTE` - Set to `1` or `true` for remote/devcontainer mode
- `PLANNOTATOR_PORT` - Fixed port to use (default: random locally, 19432 for remote)
- `PLANNOTATOR_BROWSER` - Custom browser to open plans in
- `PLANNOTATOR_SHARE_URL` - Custom base URL for share links (default: https://share.plannotator.ai)
- `PLANNOTATOR_PASTE_URL` - Base URL of the paste service API (default: https://plannotator-paste.plannotator.workers.dev)

All endpoints run on `http://localhost:{port}` where the port is either random (local) or fixed (remote mode).

## Get Plan Content

<RequestExample>
```bash curl
curl http://localhost:19432/api/plan
```

</RequestExample>

<ResponseExample>
```json 200

  "sharingEnabled": true,
  "shareBaseUrl": "https://share.plannotator.ai",
  "pasteApiUrl": "https://plannotator-paste.plannotator.workers.dev",
  "repoInfo": {
    "url": "https://github.com/user/repo",
    "owner": "user",
    "repo": "repo",
    "branch": "main"
  },

  "versionInfo": {
    "version": 3,
    "totalVersions": 5,
    "project": "my-project"
  }

```
</ResponseExample>

Retrieves the current plan content along with metadata.

</ResponseField>

  Origin identifier ("claude-code" or "opencode")
</ResponseField>

  Current permission mode (Claude Code only)
</ResponseField>

<ResponseField name="sharingEnabled" type="boolean">
  Whether URL sharing is enabled
</ResponseField>

<ResponseField name="shareBaseUrl" type="string">
  Base URL for share links
</ResponseField>

<ResponseField name="pasteApiUrl" type="string">
  Base URL of the paste service API
</ResponseField>

<ResponseField name="repoInfo" type="object">
  Git repository information
  <Expandable title="properties">
    <ResponseField name="url" type="string">Repository URL</ResponseField>
    <ResponseField name="owner" type="string">Repository owner</ResponseField>
    <ResponseField name="repo" type="string">Repository name</ResponseField>
    <ResponseField name="branch" type="string">Current branch</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="previousPlan" type="string | null">
  Previous version of the plan (if version > 1)
</ResponseField>

<ResponseField name="versionInfo" type="object">
  Version history metadata
  <Expandable title="properties">
    <ResponseField name="version" type="number">Current version number</ResponseField>
    <ResponseField name="totalVersions" type="number">Total number of versions</ResponseField>
    <ResponseField name="project" type="string">Project name</ResponseField>
  </Expandable>
</ResponseField>

## Approve Plan

<RequestExample>
```bash curl
curl -X POST http://localhost:19432/api/approve \
  -H "Content-Type: application/json" \
  -d '{
    "feedback": "Great plan! Just a minor suggestion on error handling.",
    "agentSwitch": "code-expert",
    "permissionMode": "allow",
    "planSave": {
      "enabled": true,
      "customPath": "/path/to/custom/location"
    },
    "obsidian": {
      "vaultPath": "/path/to/vault",
      "plan": "# Plan content...",
      "subfolder": "plans"
    },
    "bear": {
      "plan": "# Plan content...",
      "tags": ["plannotator", "review"]
    }
  }'
```
</RequestExample>

<ResponseExample>
```json 200
{
  "ok": true,
  "savedPath": "/home/user/.plannotator/plans/feature-implementation-2026-03-03-approved.md"
}
```
</ResponseExample>

Approves the plan and optionally saves it with annotations.

<ParamField body="feedback" type="string">
  Optional feedback/annotations from the user
</ParamField>

<ParamField body="agentSwitch" type="string">
  Agent to switch to after approval (OpenCode only)
</ParamField>

<ParamField body="permissionMode" type="string">
  Permission mode to preserve ("allow", "deny", "ask")
</ParamField>

<ParamField body="planSave" type="object">
  Plan save settings
  <Expandable title="properties">
    <ParamField body="enabled" type="boolean">Whether to save the plan</ParamField>
    <ParamField body="customPath" type="string">Custom save location</ParamField>
  </Expandable>
</ParamField>

<ParamField body="obsidian" type="object">
  Obsidian integration settings
  <Expandable title="properties">
    <ParamField body="vaultPath" type="string" required>Path to Obsidian vault</ParamField>
    <ParamField body="plan" type="string" required>Plan markdown content</ParamField>
    <ParamField body="subfolder" type="string">Subfolder within vault</ParamField>
  </Expandable>
</ParamField>

<ParamField body="bear" type="object">
  Bear notes integration settings
  <Expandable title="properties">
    <ParamField body="plan" type="string" required>Plan markdown content</ParamField>
    <ParamField body="tags" type="array">Array of tags to apply</ParamField>
  </Expandable>
</ParamField>

<ResponseField name="ok" type="boolean">
  Whether the approval was successful
</ResponseField>

<ResponseField name="savedPath" type="string">
  File path where the plan was saved (if planSave enabled)
</ResponseField>

<RequestExample>
```bash curl
curl -X POST http://localhost:19432/api/deny \
  -H "Content-Type: application/json" \
  -d '{
    "feedback": "This approach won't work because of X. Please revise to address Y.",
```

</RequestExample>

```json 200

  "ok": true,
  "savedPath": "/home/user/.plannotator/plans/feature-implementation-2026-03-03-denied.md"
}
```
</ResponseExample>

Denies the plan with feedback for revision.

<ParamField body="feedback" type="string">
  Feedback explaining why the plan was denied
</ParamField>

<ParamField body="planSave" type="object">
  Plan save settings
  <Expandable title="properties">
    <ParamField body="enabled" type="boolean">Whether to save the plan</ParamField>
    <ParamField body="customPath" type="string">Custom save location</ParamField>
  </Expandable>
</ParamField>

<ResponseField name="ok" type="boolean">
  Whether the denial was successful
</ResponseField>

<ResponseField name="savedPath" type="string">
  File path where the plan was saved (if planSave enabled)
</ResponseField>

## Save to Notes

```bash curl
curl -X POST http://localhost:19432/api/save-notes \
  -H "Content-Type: application/json" \
  -d '{
    "obsidian": {
      "vaultPath": "/path/to/vault",

      "subfolder": "plans"
    },
    "bear": {
      "plan": "# Plan content...",
      "tags": ["plannotator", "review"]
    }
  }'
```
</RequestExample>

<ResponseExample>
```json 200

  "ok": true,
  "results": {
    "obsidian": {
      "success": true,
      "path": "/path/to/vault/plans/feature-implementation.md"
    },
    "bear": {
      "success": true
    }
  }
}
```
</ResponseExample>

Saves the plan to Obsidian and/or Bear notes (decoupled from approve/deny).

<ParamField body="obsidian" type="object">
  Obsidian integration settings
  <Expandable title="properties">
    <ParamField body="vaultPath" type="string" required>Path to Obsidian vault</ParamField>
    <ParamField body="plan" type="string" required>Plan markdown content</ParamField>
    <ParamField body="subfolder" type="string">Subfolder within vault</ParamField>
  </Expandable>
</ParamField>

<ParamField body="bear" type="object">
  Bear notes integration settings
  <Expandable title="properties">
    <ParamField body="plan" type="string" required>Plan markdown content</ParamField>
    <ParamField body="tags" type="array">Array of tags to apply</ParamField>
  </Expandable>
</ParamField>

<ResponseField name="ok" type="boolean">
  Whether the save was successful
</ResponseField>

<ResponseField name="results" type="object">
  Results from each integration
  <Expandable title="properties">
    <ResponseField name="obsidian" type="object">
      <Expandable title="properties">
        <ResponseField name="success" type="boolean">Whether Obsidian save succeeded</ResponseField>
        <ResponseField name="path" type="string">Path where file was saved</ResponseField>
        <ResponseField name="error" type="string">Error message if failed</ResponseField>
      </Expandable>

    <ResponseField name="bear" type="object">
      <Expandable title="properties">
        <ResponseField name="success" type="boolean">Whether Bear save succeeded</ResponseField>
        <ResponseField name="error" type="string">Error message if failed</ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

## Get Linked Document

<RequestExample>
```bash curl
curl "http://localhost:19432/api/doc?path=docs/architecture.md"
```
</RequestExample>

<ResponseExample>
```json 200
{
  "markdown": "# Architecture\n\nThis document describes...",
  "filepath": "/home/user/project/docs/architecture.md"
}
```

```json 404
{
  "error": "File not found: docs/architecture.md"
}
```

```json 400
{
  "error": "Ambiguous filename 'README.md': found 3 matches",
  "matches": [
    "docs/README.md",
    "src/README.md",
    "tests/README.md"
  ]
}
```
</ResponseExample>

Serves a linked markdown document from the project.

<ParamField query="path" type="string" required>
  Path to the markdown file (absolute, relative to project root, or bare filename)
</ParamField>

<ResponseField name="markdown" type="string">
  The markdown content of the file
</ResponseField>

<ResponseField name="filepath" type="string">
  Resolved absolute path to the file
</ResponseField>

<ResponseField name="error" type="string">
  Error message if the file was not found or ambiguous
</ResponseField>

<ResponseField name="matches" type="array">
  List of matching files (when filename is ambiguous)
</ResponseField>

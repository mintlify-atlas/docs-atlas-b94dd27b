---
title: Annotate Server API
description: API endpoints for the markdown annotation server
---

The Annotate Server provides endpoints for annotating arbitrary markdown files. It reuses the plan editor UI in a special "annotate" mode, allowing users to provide feedback on any markdown document.

## Server Configuration

The annotate server uses the following environment variables:

- `PLANNOTATOR_REMOTE` - Set to `1` or `true` for remote/devcontainer mode
- `PLANNOTATOR_PORT` - Fixed port to use (default: random locally, 19432 for remote)

All endpoints run on `http://localhost:{port}` where the port is either random (local) or fixed (remote mode).

## Get Markdown Content

<RequestExample>
```bash curl
curl http://localhost:19432/api/plan
```
</RequestExample>

<ResponseExample>
```json 200
{
  "plan": "# Architecture Documentation\n\n## Overview\n\nThis document describes the system architecture...",
  "origin": "opencode",
  "mode": "annotate",
  "filePath": "/home/user/project/docs/architecture.md",
  "sharingEnabled": true,
  "shareBaseUrl": "https://share.plannotator.ai",
  "repoInfo": {
    "url": "https://github.com/user/repo",
    "owner": "user",
    "repo": "repo",
    "branch": "main"
  }
}
```
</ResponseExample>

Retrieves the markdown content to be annotated.

Note: This endpoint reuses `/api/plan` from the plan server but includes `mode: "annotate"` to signal the UI to operate in annotation mode.

<ResponseField name="plan" type="string" required>
  The markdown content of the file to annotate
</ResponseField>

<ResponseField name="origin" type="string">
  Origin identifier ("claude-code" or "opencode")
</ResponseField>

<ResponseField name="mode" type="string" required>
  Server mode - always "annotate" for this server
</ResponseField>

<ResponseField name="filePath" type="string" required>
  Original file path being annotated
</ResponseField>

<ResponseField name="sharingEnabled" type="boolean">
  Whether URL sharing is enabled
</ResponseField>

<ResponseField name="shareBaseUrl" type="string">
  Base URL for share links
</ResponseField>

<ResponseField name="repoInfo" type="object">
  Git repository information
  <Expandable title="properties">
    <ResponseField name="url" type="string">Repository URL</ResponseField>
    <ResponseField name="owner" type="string">Repository owner</ResponseField>
    <ResponseField name="repo" type="string">Repository name</ResponseField>
    <ResponseField name="branch" type="string">Current branch</ResponseField>
  </Expandable>
</ResponseField>

## Submit Annotation Feedback

<RequestExample>
```bash curl
curl -X POST http://localhost:19432/api/feedback \
  -H "Content-Type: application/json" \
  -d '{
    "feedback": "This architecture doc needs clarification on:\n\n1. Database schema design\n2. API authentication flow\n3. Caching strategy",
    "annotations": [
      {
        "id": "ann-1",
        "blockId": "block-5",
        "type": "COMMENT",
        "startOffset": 0,
        "endOffset": 42,
        "text": "Add sequence diagram here to illustrate the flow",
        "originalText": "The authentication process works as follows",
        "createdAt": 1709481600000
      },
      {
        "id": "ann-2",
        "blockId": "block-8",
        "type": "REPLACEMENT",
        "startOffset": 10,
        "endOffset": 25,
        "text": "PostgreSQL with Redis caching",
        "originalText": "a SQL database",
        "createdAt": 1709481650000
      },
      {
        "id": "ann-3",
        "blockId": "block-12",
        "type": "DELETION",
        "startOffset": 0,
        "endOffset": 58,
        "originalText": "This section is outdated and should be removed entirely.",
        "createdAt": 1709481700000
      },
      {
        "id": "ann-4",
        "blockId": "global",
        "type": "GLOBAL_COMMENT",
        "startOffset": 0,
        "endOffset": 0,
        "text": "Overall the document is well-structured but needs technical depth",
        "originalText": "",
        "createdAt": 1709481750000,
        "images": [
          {
            "path": "/tmp/plannotator-uploads/diagram.png",
            "name": "proposed-architecture"
          }
        ]
      }
    ]
  }'
```
</RequestExample>

<ResponseExample>
```json 200
{
  "ok": true
}
```

```json 500
{
  "error": "Failed to process feedback"
}
```
</ResponseExample>

Submits annotation feedback to the agent session.

<ParamField body="feedback" type="string" required>
  Annotation feedback text to send to the agent
</ParamField>

<ParamField body="annotations" type="array">
  Array of annotation objects from the annotation UI
  <Expandable title="annotation object">
    <ParamField body="id" type="string" required>Unique annotation ID</ParamField>
    <ParamField body="blockId" type="string" required>ID of the markdown block being annotated</ParamField>
    <ParamField body="type" type="string" required>
      Annotation type:
      - `COMMENT` - Add a comment/suggestion
      - `REPLACEMENT` - Replace text with new text
      - `DELETION` - Mark text for deletion
      - `INSERTION` - Insert new text
      - `GLOBAL_COMMENT` - Document-level comment
    </ParamField>
    <ParamField body="startOffset" type="number" required>Start character offset in the block</ParamField>
    <ParamField body="endOffset" type="number" required>End character offset in the block</ParamField>
    <ParamField body="text" type="string">Annotation text (for COMMENT, REPLACEMENT, INSERTION)</ParamField>
    <ParamField body="originalText" type="string" required>The selected/original text</ParamField>
    <ParamField body="createdAt" type="number">Unix timestamp in milliseconds</ParamField>
    <ParamField body="author" type="string">Author identity</ParamField>
    <ParamField body="images" type="array">
      <Expandable title="image attachment">
        <ParamField body="path" type="string">Temporary file path</ParamField>
        <ParamField body="name" type="string">Human-readable image label</ParamField>
      </Expandable>
    </ParamField>
  </Expandable>
</ParamField>

<ResponseField name="ok" type="boolean">
  Whether the feedback was successfully received
</ResponseField>

<ResponseField name="error" type="string">
  Error message if the operation failed
</ResponseField>

## Serve Image

<RequestExample>
```bash curl
curl "http://localhost:19432/api/image?path=/tmp/plannotator-uploads/diagram.png"
```
</RequestExample>

<ResponseExample>
```text 200
[Binary image data]
```

```text 404
File not found
```

```text 403
Access denied: path is outside allowed directories
```
</ResponseExample>

Serves an image from a local path or temporary upload directory.

<ParamField query="path" type="string" required>
  Path to the image file
</ParamField>

## Upload Image

<RequestExample>
```bash curl
curl -X POST http://localhost:19432/api/upload \
  -F "file=@architecture-diagram.png"
```
</RequestExample>

<ResponseExample>
```json 200
{
  "path": "/tmp/plannotator-uploads/c3d4e5f6-a7b8-9012-cdef-123456789012.png",
  "originalName": "architecture-diagram.png"
}
```

```json 400
{
  "error": "No file provided"
}
```

```json 400
{
  "error": "Invalid file type. Allowed: jpg, jpeg, png, gif, webp, svg"
}
```
</ResponseExample>

Uploads an image file to a temporary directory for attachment to annotations.

<ParamField body="file" type="file" required>
  Image file to upload (multipart/form-data)
</ParamField>

<ResponseField name="path" type="string">
  Temporary file path where the image was saved
</ResponseField>

<ResponseField name="originalName" type="string">
  Original filename of the uploaded image
</ResponseField>

<ResponseField name="error" type="string">
  Error message if upload failed
</ResponseField>

## Annotation Types

The annotate server supports five types of annotations:

### COMMENT
Add a comment or suggestion without modifying the text.

```json
{
  "type": "COMMENT",
  "originalText": "selected text",
  "text": "Your comment here"
}
```

### REPLACEMENT
Replace selected text with new text.

```json
{
  "type": "REPLACEMENT",
  "originalText": "old text",
  "text": "new text"
}
```

### DELETION
Mark text for deletion.

```json
{
  "type": "DELETION",
  "originalText": "text to delete"
}
```

### INSERTION
Insert new text at a position.

```json
{
  "type": "INSERTION",
  "originalText": "context text",
  "text": "text to insert"
}
```

### GLOBAL_COMMENT
Document-level comment not tied to specific text.

```json
{
  "type": "GLOBAL_COMMENT",
  "text": "Overall document feedback",
  "images": [
    {
      "path": "/tmp/path.png",
      "name": "diagram-label"
    }
  ]
}
```
